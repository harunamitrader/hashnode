> Source: github OpenSpace

Claude Code や Codex、OpenClaw、nanobot など任意のエージェントに「自己進化」を付け足す基盤が **OpenSpace**。失敗を自動修復し、成功パターンを学習・共有していくことで、トークン消費を抑えつつ仕事の品質を上げることを狙っています。

## OpenSpace が足す3つの力

- **Self-Evolution**: Auto-fix / Auto-improve / Auto-learn。スキルの実行ログを監視し、壊れたら即修復、成功パターンは新バージョンとして残す。
- **Collective Intelligence**: 改善済みスキルをクラウドにアップロード/ダウンロード。公開・非公開・チーム限定を選べる。
- **Token Efficiency**: 同じ課題に毎回ゼロから推論しない。実績を再利用して 46% トークン削減 & 4.2x 収益向上（GDPVal ベンチ、ClawWork 比）。

## クイックスタート

```bash
git clone https://github.com/HKUDS/OpenSpace.git && cd OpenSpace
pip install -e .
openspace-mcp --help
```

クローンが重い場合は `--filter=blob:none --sparse` で `assets/` を除外可能。

### Path A: 既存エージェントに組み込む

1) MCP 設定に OpenSpace を追加（例: Claude Code/Codex の `mcpServers`）  
   - `command: openspace-mcp`  
   - `OPENSPACE_HOST_SKILL_DIRS`, `OPENSPACE_WORKSPACE` を環境変数で指定  
   - 必要なら `OPENSPACE_API_KEY`（クラウド共有用）  
2) `openspace/host_skills/delegate-task` と `skill-discovery` を自分の `skills/` にコピー。  
3) これでエージェントは「いつ OpenSpace を呼ぶか」を理解し、自己進化サイクルを回し始める。

### Path B: Co-worker として使う

`openspace-mcp` を直接 CLI から動かし、タスク実行・進捗通知・ローカルダッシュボード（`showcase/README.md`）を使うモード。個人開発でもそのまま活用できる。

## 仕組みのざっくり像

- 実行ログを監視し、失敗時は自動修復タスクを起動、成功パターンは技能として保存。  
- スキル性能・エラー率をメトリクス化し、しきい値で自動リグレッション検知。  
- クラウドコミュニティ（`open-space.cloud`）に登録すれば、改善済みスキルを共有・取得できる。

## ベンチマーク: GDPVal

- 50件の業務タスクで **4.2× 収益 / 46% トークン削減**（ClawWork ベースライン、Qwen3.5-Plus）。  
- カテゴリ別でもすべて改善（例: Compliance +18.5%、Engineering +8.7%、Docs はトークン 56% 減）。

## 使う前の注意

- Python 3.12+ 前提。`assets/` が大きいので回線が細い場合は sparse clone 推奨。  
- クラウド共有は任意。機密スキルは非公開/チーム限定で管理する。  
- 自動修復/進化は実行ログを収集するため、扱うデータの秘匿性に注意。  
- 依存する外部ツール（ffmpeg など）は各スキルごとに確認を。

## まとめ

OpenSpace は「一度できたことを全エージェントで共有し、失敗から自動で学ぶ」ための自己進化エンジンです。  
`pip install -e .` → MCP 追加 → スキルコピーの 3 ステップで既存エージェントに組み込めるので、まずは Path A から導入してトークン削減と安定化の手応えを試してみてください。
