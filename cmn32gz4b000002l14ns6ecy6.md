スマートグラス向け AI は話題になりやすい一方で、実際に「何ができて、どこまで開いているのか」は見えにくいことが多いです。VisionClaw は、その中身をかなり具体的に見せてくれるリポジトリでした。

## VisionClaw は何か

VisionClaw は、Meta Ray-Ban スマートグラス向けのリアルタイム AI アシスタント実装です。グラスやスマホのカメラ映像とマイク音声を Gemini Live API に送り、会話を返しつつ、必要に応じて OpenClaw 経由で外部アクションまで実行できます。

要するに、「AI が今見えているものを理解し、その場で話し、必要なら何かを実行する」までを一つの流れにした OSS です。

## 何ができるのか

README の例はかなり分かりやすく、次のような体験を想定しています。

- 「いま何を見ている？」と聞いて、視界の内容を説明させる
- 「牛乳を買い物リストに追加して」と頼み、OpenClaw 経由で実行する
- 「John に遅れると送って」と頼み、メッセージ送信に回す
- 「近くの良いコーヒー店を探して」と頼み、検索結果を音声で返させる

ここで重要なのは、会話だけで終わらず、OpenClaw を足すと実行系に広がることです。VisionClaw 単体でも視覚付き音声アシスタントとして成立しますが、OpenClaw をつなぐと「行動するアシスタント」に近づきます。

## 目立つポイント

### 1. グラスがなくても試せる

この手のプロジェクトは、専用ハードウェアがないと触れないことが多いです。VisionClaw はそこが比較的やさしく、iPhone や Android のカメラを使う Phone mode が用意されています。

つまり、Ray-Ban Meta を持っていなくても、パイプライン自体の体験や開発は始められます。これはかなり大きいです。

### 2. Gemini Live を中心に据えた構成が分かりやすい

映像は低頻度の JPEG フレーム、音声はリアルタイムの双方向ストリームとして Gemini Live API に送られます。リポジトリの説明からは、STT を挟んだ後付け構成というより、「最初から会話セッションとして扱う」設計であることが読み取れます。

このため、単なる画像認識アプリではなく、視覚文脈を持つ会話インターフェースとして見たほうが実態に近いです。

### 3. OpenClaw との接続で用途が一気に広がる

OpenClaw は必須ではありませんが、VisionClaw の面白さをかなり押し上げています。ツール呼び出しはアプリ側で受け、OpenClaw Gateway に流す構成になっているため、Web 検索、メッセージ、ノート、リマインダー、スマートホームなどへ拡張しやすいです。

「グラスで見たものについて話せる」から、「グラス経由で指示して何かを動かせる」へ進めるのがこの構成の強みです。

### 4. WebRTC で視点共有まで入っている

さらに面白いのは、ブラウザ向けのライブ共有も用意されていることです。グラス視点をリアルタイムで共有できるので、単独利用だけでなく、デモや遠隔サポートの土台にもなります。

## どう動くのか

高レベルには、次の流れです。

1. グラスまたはスマホから映像と音声を取得する
2. アプリが Gemini Live API に会話セッションを張る
3. Gemini が視覚文脈を踏まえて返答する
4. ツール呼び出しが必要なら OpenClaw Gateway に渡す
5. 実行結果を受けて、Gemini が最終応答を話す

この分離はかなり筋がいいです。VisionClaw 自体はウェアラブル側の体験に集中し、外部サービス操作は OpenClaw に寄せています。

## 誰に向いているか

次のような人にはかなり刺さります。

- ウェアラブル AI の実装例を探している人
- Gemini Live API を実際のプロダクト感ある形で触りたい人
- OpenClaw の実運用イメージを広げたい人
- Ray-Ban Meta グラスを開発プラットフォームとして見たい人

特に、マルチモーダル AI を「スマホの中だけ」で終わらせず、現実世界の視点入力に接続したい人には良い題材です。

## 注意点

- Gemini API キーは必須です
- Android は DAT SDK の取得に GitHub Packages 用トークンが必要です
- OpenClaw を入れない場合、実行系の強みはかなり減ります
- グラスがなくても試せますが、本来の魅力はウェアラブル前提の UX にあります
- README の clone 例が旧リポジトリ URL のままで、細部のメンテは完全には追いついていないようです

ここは事実と推測を分けて見るべきです。事実として、iOS/Android 両対応、Gemini Live、OpenClaw 連携、WebRTC 共有まで入っています。推測としては、これは完成品アプリというより、ウェアラブル AI を試作・研究するための優秀な土台です。
