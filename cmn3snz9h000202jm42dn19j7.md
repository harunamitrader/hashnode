Anthropic は 2026年3月23日に、Claude の新しい使い方として `Dispatch` と `Computer use` をまとめて発表しました。

一言でいうと、`スマホから Claude に仕事を投げて、必要なら自分のPCを Claude に触らせて進めさせる` 方向の機能です。

いままでの AI は、チャット欄で答えを返すところまでが中心でした。今回の発表で見えてきたのは、`自分のデスクトップ環境で実際に作業する Claude` です。

## まず Dispatch は何か

Dispatch は、Claude Cowork と Claude Code で使える、`スマホとデスクトップをまたいで同じ会話を続ける` 仕組みです。

たとえば、通勤中にスマホから Claude にタスクを投げておいて、あとでデスクトップを開いたときに結果を受け取る、といった使い方ができます。Anthropic のヘルプでは、Claude が状況に応じて Cowork か Claude Code の適切なセッションを立ち上げ、結果をスレッドに返す形で説明されています。

ポイントは、ただスマホからメッセージを送れるだけではないことです。Dispatch は `手元のPCで動く Claude` に仕事を回す入口になっています。ローカルファイル、設定済みのコネクタ、プラグイン、そして後述する Computer use とつながることで、「その場で答えるAI」より一段進んだ役割を持ちます。

## Computer use は何か

Computer use は、Claude が必要に応じて `画面を見て、クリックして、入力して、アプリを開いて作業する` 機能です。

公式説明では、Claude はまず Slack や Google Drive のようなコネクタを優先します。コネクタがない場合に、ブラウザ操作や画面操作へ進みます。つまり、最初から何でも画面経由でゴリ押しするのではなく、`使える統合があればそちらを先に使う` 設計です。

これが入ることで、Claude は次のような仕事をしやすくなります。

- ブラウザ上の社内ダッシュボードを開いて必要な情報を拾う
- ローカルファイルを見ながらレポートをまとめる
- IDE や開発ツールを開いて、修正や確認を進める
- スプレッドシートや特定アプリを操作して定型作業を進める

要するに、`コネクタがないから無理` で止まりにくくなるのが大きいです。

## この発表の新しさ

今回の発表で面白いのは、Dispatch と Computer use が `別々の小機能` ではなく、ひとつの流れとしてつながっていることです。

スマホから Claude にタスクを渡す。Claude は必要ならデスクトップ側で Cowork や Claude Code のセッションを起動する。そこでファイルやコネクタを使い、それでも足りなければ画面操作に降りていく。終わったら結果を返す。

この流れがあるので、今回の新機能は単なる「AIがブラウザを触れます」ではありません。むしろ、`AIに実務を持たせる導線が一段つながった` と見る方が実態に近いです。

特に Claude Code と組み合わさると、コードを書くアシスタントというより、`開発環境ごと任せる余地が広がった` ことになります。ブログでも、IDE を触る、テストを回す、PR までつなげる、といったイメージが示されています。

## いま使える範囲

2026年3月24日時点で確認できる範囲では、次の理解が正確です。

- この発表記事の公開日は 2026年3月23日
- Dispatch は Claude Cowork での research preview 機能として案内されている
- Computer use も research preview
- どちらも Pro / Max プラン向け
- Dispatch 自体はデスクトップアプリとモバイルアプリの組み合わせが前提
- Computer use は現時点では macOS の Claude Desktop アプリで提供
- Windows は Dispatch の要件説明には出てくるが、Computer use は `coming soon` 扱い

このあたりは少し混同しやすいので、`Dispatch と Computer use は近いが同じものではない` と分けて理解した方がよさそうです。

## 制約とリスクもかなり大きい

便利そうに見えますが、公式の安全ガイドはかなり強めです。

理由は単純で、Claude が `実際のデスクトップ環境` を触るからです。ヘルプでは、Computer use は Cowork の通常の仮想マシン外で動くと説明されています。つまり、隔離された実験環境ではなく、ふだん使っているアプリや画面に近い場所で動く、ということです。

そのため、公式には次のような注意が並んでいます。

- アプリごとの許可が必要
- 機密性の高いアプリには使わない方がいい
- 金融、医療、法務などの情報は避けるべき
- prompt injection や誤操作のリスクは残る
- 複雑なタスクは失敗したり、やり直しが必要になることがある

つまり、これは `何でも安心して自動で任せられる完成品` ではありません。便利な方向性はかなり見えていますが、現時点では `強いが慎重に使うべき research preview` です。

## どう見るべきか

この発表は、Claude がまた少し「チャットAI」から離れた、という意味で大きいです。

これまでの流れでは、AI が文章を書く、コードを書く、検索する、という段階が中心でした。Dispatch と Computer use では、そこに `自分の環境で実行する` が入ってきます。

もちろん、まだ荒いし危ない部分もあります。ですが、`スマホから仕事を投げる`、`PC側で処理を進める`、`足りない部分は画面操作で埋める` という形は、今後の AI エージェントの方向性をかなり分かりやすく示しています。

Claude の新機能というより、`AI がどこまで人の作業環境に入ってくるのか` を考える上で見ておく価値のある発表でした。
