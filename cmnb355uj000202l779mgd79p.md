> Source: github TREK

**TREK** は自ホストで動くリアルタイム旅行プランナー。地図・日程・予約・予算・持ち物・ファイル管理を一つにまとめ、仲間と同時編集できます。PWA なのでスマホにインストールしてオフラインでも使えます。

## 主な特徴

- **日程作成**: ドラッグ&ドロップで日単位プランを組み、順序最適化→Google Maps へエクスポート。
- **地図と検索**: Leaflet 地図、写真ピン、クラスタ表示。検索は Google Places (写真/評価/営業時間) か OSM（APIキー不要）を選択。
- **旅の管理**: 予約(フライト/ホテル/レストラン)と添付ファイル、カテゴリ別予算、マルチ通貨、パッキングリスト、PDFエクスポート。
- **コラボ**: WebSocket でリアルタイム同期、マルチユーザー招待、ロール権限、チャット/投票/参加管理、OIDC SSO。
- **PWA/オフライン**: Workbox で地図タイル・APIレスポンス・アップロードをキャッシュ。ホーム画面追加でネイティブ風に動作。
- **アドオン**: Vacay(休暇計画+祝日/社休日)、Atlas(訪問国マップ/統計)、Collab(チャット等)、通貨換算や時計のウィジェットを管理画面でON/OFF。
- **管理機能**: 多言語(EN/DE/zh-Hans/NL/RU)、ダークモード、バックアップスケジュール、リリース履歴、カスタム地図タイル。

## クイックスタート

最短は Docker 1 行:
```bash
docker run -d -p 3000:3000 -v ./data:/app/data -v ./uploads:/app/uploads mauriceboe/trek
```
初回登録ユーザーが管理者になります。  
本番は Docker Compose + 逆プロキシ(HTTPS)推奨。WebSocket `/ws` のアップグレードを許可してください。

## PWAとして使う

1) インスタンスを開く（HTTPS 推奨）  
2) iOS: 共有→「ホーム画面に追加」 / Android: メニュー→「アプリをインストール」  
3) フルスクリーン＋独自アイコンで起動、オフラインキャッシュが有効。

## 技術スタック

- バックエンド: Node.js 22 + Express + SQLite(better-sqlite3)  
- フロント: React 18 + Vite + Tailwind  
- リアルタイム: WebSocket (ws)  
- PWA: vite-plugin-pwa + Workbox  
- 認証: JWT + OIDC  
- 地図: Leaflet, react-leaflet-cluster  
- 天気: Open-Meteo (無料/キー不要)

## だれに向いているか

- 家族や友人と旅行プランを一緒に作りたいが、外部SaaSではなく自分でホストしたい人  
- オフラインでも日程と地図を確認したい人  
- 予約情報・ファイル・予算・パッキングを一元管理したい人

## 注意点

- Google Places 検索を使う場合は API Key が必要（OSM なら不要）。  
- 50MB 超のファイル添付は不可（デフォルト）。  
- 逆プロキシ設定時は WebSocket `/ws` を通すこと、TLS 証明書の更新を忘れないこと。

## まとめ

TREK は「自分たちだけの旅行プランナー」を作るのに十分な機能を揃えたオープンソース PWA です。まずは Docker で立ち上げ、ライブデモ https://demo-nomad.pakulat.org で操作感を掴んでから、必要なアドオンや地図設定を好みに合わせて有効化してみてください。
